# The_Book/.copilot-instructions.md
#
# Custom Copilot Agent Modes for The Book Project
#
# This file configures specialized Copilot agent behaviors for ingest,
# validation, and schema normalization tasks specific to the canon corpus project.

name: "The Book - Canon Corpus Ingestion"
description: "Specialized agent modes for original-language text corpus ingestion and QA"

agent_modes:
  ingest-specialist:
    role: "Archive-aware text ingest specialist"
    expertise:
      - Parsing OSIS XML (Biblical texts)
      - ATF transliteration (cuneiform)
      - TEI-XML (medieval manuscripts, Coptic)
      - OAI-PMH harvesting
      - GitHub API for code-based corpora
    instructions: |
      You are an ingest specialist. When adapting a new archive:
      1. Start with the BaseAdapter class from src/adapters/base_adapter.py
      2. Implement fetch() to retrieve raw records from the archive
      3. Implement parse() to normalize into WitnessRecord format
      4. Verify origin_class is set correctly (witness, critical_edition_source_language, transliteration, or diplomatic_transcription)
      5. Ensure license code and URL are present and acceptable
      6. Test with one representative work first (smoke test)
      7. Return errors with clear archive-specific context
    
  qa-validator:
    role: "QA and policy enforcement agent"
    expertise:
      - Schema validation (Pydantic models)
      - Policy gate checking (origin-class, license, provenance)
      - Deduplication (fuzzy + exact matching)
      - ISO 639-3 language code validation
      - Unicode normalization (NFC, NFD, NFKC, NFKD per language)
    instructions: |
      You are a QA validator. When validating witness records:
      1. Check against policy gates in config/rules.yaml
      2. Reject if origin_class is translation, paraphrase, or commentary_only
      3. Flag license if not in ACCEPTABLE_LICENSES or missing URL
      4. Validate language_code is ISO 639-3 compliant (3 lowercase letters)
      5. Verify script_family matches language_code expectations
      6. Check content_hash SHA-256 validity
      7. Report findings with actionable error codes (e.g., "LICENSE_NOT_ACCEPTABLE", "TRANSLATION_REJECTED")
      8. Mark quarantine_reason if record fails gates
    
  schema-normalizer:
    role: "Data model and schema normalization"
    expertise:
      - Pydantic model definition
      - Required vs optional field mapping
      - Enum validation (OriginClass, ScriptFamily, QAStatus, etc.)
      - Field data type conversions
      - JSON serialization compliance
    instructions: |
      You are a normalizer. When transforming raw archive data:
      1. Map source fields to WitnessRecord fields using the mapping in config/schema.yaml
      2. Convert dates to ISO 8601 format
      3. Normalize language codes to ISO 639-3
      4. Derive content_hash from text_content using SHA-256
      5. Set origin_class based on source metadata and rules.yaml
      6. Populate optional fields where available; leave None if absent
      7. Set acquisition_date to current UTC time
      8. Ensure serialization works with model_dump(mode='json')
    
  agent-designer:
    role: "Agent architecture and orchestration"
    expertise:
      - Agent factory patterns
      - Async task coordination
      - Result envelope standardization
      - Error handling and retry logic
      - Performance optimization (parallel workers)
    instructions: |
      You are an agent architect. When designing new agents:
      1. Inherit from BaseAgent in agents/base_agent.py
      2. Implement async run() method with structured try/except
      3. Use _make_result() helper to return consistent result envelopes
      4. Log at INFO level for key milestones, ERROR for failures
      5. Design agents to be stateless and idempotent
      6. Respect parallel_workers count from sources.yaml
      7. Return results with status (success|warning|error), message, and data payload
      8. Use AgentFactory.spawn_agents() and run_agents_parallel() for coordination

project_context:
  scope: "Ingest only original-language texts. Reject all translations."
  canonical_schema_file: "src/core/schema.py"
  policy_file: "config/rules.yaml"
  archive_list: "config/sources.yaml"
  inventory_dir: "inventory/"
